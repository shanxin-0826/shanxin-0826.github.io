模块化

早期的网页中，是没有一个实质的模块规范的;​

实现模块化的方式，就是最原始的通过script标签来引入多个js文件;

问题：

1.无法选择要引入模块的指定内容;

2.在复杂的模块场景下非常容易出错;

于是，我们就急需在js中引入一个模块化的解决方案;

CommonJS​

在node中，默认支持的模块化规范叫做CommonJS;

在CommonJS中，一个js文件就是一个模块;

CommonJS规范​

引入模块

1.使用require('模块的路径')函数来引入模块;

2.引入自定义模块时;

    模块名要以 ./ 或 .// 开头;

3.扩展名可以省略;

    在Common.js中，如果省略js文件的扩展名;

    node会自动为文件补全扩展名;

    查找顺序：

        js --> json --> node(特殊);

4.引入核心模块时;

    直接写核心模块的名字即可;

    也可以在核心模块前添加 node：;

5.cjs为扩展名，表示是一个CommonJS标准的模块;

6.文件夹作为模块时;

    文件夹中必须有一个模块的主文件;

    node会自动寻找index.js作为主文件;

7.可以通过对象属性或解构赋值引入module.exports的单个值;

定义模块

1.在定义模块时，模块中的内容默认是不能被外部看到的;

可以通过exports来设置要向外部暴露的内容;

2.访问exports的方式有两种:

    exports;

    module.exports;

3.当我们在其他模块中引入当前模块时;

require函数返回的就是exports;

4.我们将希望暴露给外部模块的内容设置为exports的属性;

5.可以通过exports一个一个的导出值;

也可以直接通过module.exports同时导出多个值;

模块的包装（原理）​

所有的CommonJS的模块都会被包装到一个函数中;

( function ( exports, require, module, __filename, __dirname){

    模块的代码会被放到这里;

});

exports，用来设置模块向外部暴露的内容;

require，用来引入模块的方法;

module，当前模块的引用;

__filename，当前模块的绝对路径;

__dirname，当前模块所在的目录路径;

ES module​

默认情况下，node中的模块化标准是CommonJS;

要想使用ES的模块化，可以采用以下两种方案：

1.使用mjs作为扩展名;

2.修改package.json将模块化规范设置为ES模块;

当我们设置 "type" : "module" ;

当前项目下所有的js文件都默认为ES module;

ES 模块化​

向外部导出内容：

export 声明 变量名 = 值;

设置默认导出，一个模块中只有一个默认导出：

export default 值;

注意：只可以导出值，不可以在default后声明变量;

向内部导入内容：

1.ES模块不能省略扩展名（官方标准）;

import { a , b , c } from "./文件名.mjs";

2.默认导入变量名称不可修改，但我们可以通过 as 来指定别名;

import { a as hello , b , c } from "./文件名.mjs";

3.可以通过*来导入所有数据，但开发时要尽量避免import * 情况;

import * as 别名 from "./文件名.mjs";

4.导入模块的默认导出，默认导出的内容可以随意命名;

import 名字 from "./文件名.mjs"

5.通过ES模块化，导入的内容都是常量（对象属性可修改）;

6.ES模块都是运行在严格模式下的;

7.ES模块化，在浏览器中同样支持;

但是通常我们不会直接使用，会结合打包工具使用;
